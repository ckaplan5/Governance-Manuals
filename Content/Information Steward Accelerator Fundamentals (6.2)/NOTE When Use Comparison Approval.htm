The users assigned to the Review role will see an enhanced Approval page to view original data along with any changes to the data entered on the Data Entry page.

  1. Close the Vertical View.
  1. Modify the Comparison Approval Stored Procedure

The stored procedure created when the Template was generated needs to be modified to display the comparison approval page.

The procedure must be modified to correctly populate table ttCustomer_Informal_WordXXCompare.

  • The ttCustomer_Informal_WordXXCompare must contain one record for each corresponding record in the ttCustomer_Informal_WordXX table
  • The records in ttCustomer_Informal_WordXCompare must contain the values currently in SAP for the data inserted into ttCustomer_Informal_WordXX.

The following are guidelines for modifying this procedure:

  • Delete any records from ttCustomer_Informal_WordXXCompare for the current Request.
  • Delete any records from ttCustomer_Informal_WordXXCompare where ID = an ID retrieved by the select statement.  When requests are automatically created by partial rejection of another request, ID values will be duplicates of IDs inserted for the original request.
  • Remove the SELECT statement delivered in this procedure.
  • Insert into ttCustomer_Informal_WordXXCompare by joining ttCustomer_Informal_WordXX with all source tables or views originally used to populate ttCustomer_Informal_WordXX.
  • Key fields should be populated from ttCustomer_Informal_WordXXCompare.
  • Data fields should be populated from the original source(s).
  • Limit the Select/Insert to the current RequestID.

To modify the stored procedure:

  1. Navigate to the cMass_Data database in SQL.
  2. Expand Programmability.
  3. Expand Stored Procedures.
  4. Locate the webCustomer_Informal_WordXXCompareIns stored procedure for the Template.
  5. Right-click the procedure.
  6. Select Modify.
  7. Enter the following replacing XX with the Training ID.
