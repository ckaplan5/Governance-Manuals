4.9 Import Column Properties and Configure Excel Columns for a Template

By default, the Data Role is configured to export and import Excel spreadsheets. This is configurable on the Vertical View of the Data Role on the Import Settings tab. At the request role level, if Excel Import is enabled for the Template used by the request, a user assigned to the role can generate an Excel file to use for data entry.

Once the Column Properties have been configured in System Administration for the Data Entry page, they can be imported to the Excel file and additional configurations can be made.

The columns display in the generated data entry spreadsheet and on the Data Entry can be controlled by either adding Column Properties or configuring it at the Template Role Level for a request based on the Template. Column access and color settings can also be configured.

Column display can be affected by settings:

  • At the template role level on the Template (Role Excel Column Control) page
  • At the template role level on the Template (Role Column) page
  • At the Review filter level if the role is Review

dspCompose™ compares these column display settings, if applicable, and uses the most restrictive column control status (set on the Template (Role Column) page) to determine if a column should be displayed. Hidden is more restrictive then Disabled. Disabled is more restrictive then Enabled.

The Template (Role Excel Column Control) page allows the Template Administrator to set:

  • Whether a column is included or excluded in the Excel file generated for data entry and on the data entry pages
  • Whether a column is read only
  • What type of control, such as a list box, is used to enter data

The Vertical View of the Template (Role Excel Column Control) page allows the user to configure further settings, such as colors, list settings, and custom validations.
