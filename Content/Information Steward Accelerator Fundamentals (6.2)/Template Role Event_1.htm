5.16 Configure Page Properties for the Data Entry Page

All pages in the DSP must have a table and a Horizontal and/or Vertical View. When the Template was generated, dspCompose™ automatically created the data entry page, table and the Horizonal View for the Material Master01 Data Entry page. The table and Horizontal View were automatically registered to the page with default Page Properties. The Page Properties can be modified to further control how the page functions.

The following are basic page properties which determine the data and functionality on the page.

  • Table – Determines where the data for the page is stored.
  • Horizontal View – Determines what fields from the table are to display on the page.
  • Vertical View – Determines if there is a Vertical View for the table displaying additional information.
  • Horizontal Menu ID – Determines what menu displays at the top of the page.
  • Vertical Menu ID – Determines what menu, if any displays at the left of the page.
  • Insert Method – Determines the behavior when adding a new record. There are 5 options:
  • Not Supported – No records can be added to the page. The Add icon on the Grid Toolbar does not display.
  • Horizontal Insert – Once a record is added and saved the current view persists.
  • Horizontal Insert/Switch to Vertical – Once the record is added and saved, the Vertical View displays. The Vertical View must be saved to save any information on the Vertical View only. If the Vertical View is not saved, the data entered on the Horizontal View persists and still displays.
  • Horizontal/Switch to Vertical – Once the record is added and saved, the Vertical View displays. The Vertical View must be saved to save any information on the Horizontal and Vertical Views. If the Vertical view is not saved, the data entered on the Horizontal View is not saved.
  • Switch to Vertical Immediately – Upon adding a record, the Vertical View displays.
  • Update Method – Determines the behavior when editing a record. There are 5 options:
  • Not Supported – No records can be added edited. The Edit icon on the Grid Toolbar does not display.
  • Horizontal Update – Once a record is edited and saved the current view persists.
  • Horizontal Update/Switch to Vertical – Once the record is edited and saved, the Vertical View displays. The Vertical View must be saved to save any information on the Vertical View only. If the Vertical View is not saved, the data edited on the Horizontal View persists and still displays.
  • Horizontal/Switch to Vertical – Once the record is edited and saved, the Vertical View displays. The Vertical View must be saved to save any information on the Horizontal and Vertical Views. If the Vertical view is not saved, the data edited on the Horizontal View is not saved.
  • Switch to Vertical Immediately – Upon editing a record, the Vertical View displays.
  • Support Delete – Determines if records on the page can be deleted. When unchecked, the Delete icon does not display.

If there are a lot of columns on the page it will be necessary to scroll to the right to view all the data. To avoid any horizontal scrolling, both the Horizontal and the Vertical should be registered to the page. The Horizontal View which was created by dspCompose should be modified to contain the basic information, A Vertical View should be created in SQL and contain the columns on the Horizontal View and all additional columns. Tabs can also be created to organize the data on the Vertical View.

By default, the add and delete functionality is available on the generated Data Entry page. Modify the Page Properties so the users assigned to the Data and Manufacturing Roles cannot add or delete on the page.

To change the Page Properties:

  1. Click Admin on the Navigation pane.
  2. Click WebApps on the Navigation pane.
  3. Locate cMass_Data WebApp.
  4. Click Pages for the cMass_Data WebApp.
  5. Enter MaterialMasterUpdateXX in the Filter field.
  6. Click the Enter key.
  7. Locate MaterialMasterUpdateXX page.
  8. Click the Vertical View .
  9. Click the Edit button.
  10. Select Not Supported from the Insert Method list box.
  11. Click Support Delete check box to disable.
  12. Click the Save button.
  13. Close the Vertical View.
