1.8 Post the Data

Once all roles have been approved by the user assigned to the Review role, the request data is ready to be posted. As a recommendation, if a large number of records are to be posted, post request data during non-peak work hours, for example, after 5:00 p.m.

NOTE:Posting options are configured at the Template level and can vary per Template. Some options may not be available for certain Template types.

Posting options determine whether a Request is posted in the background or the foreground.

A Request is posted in the background if there is a large amount of data to process that may affect system performance or may time out a session. Monitor posts that run in the background on the Request Role Finish Monitor page.

A post that runs in the foreground processes immediately and is appropriate for Requests that are not too large to time out the system.

Posting options include:

  • Post - Immediately posts data in the foreground and prompts for Back End Authentication. Back End Authentication requires the logged in user to enter a user ID and password for the target system in order to post.
  • System Credential Post - Immediately posts data in the foreground using the stored target system credentials entered on the Posting and File tab on the Parameters page. If multiple users post using the same ID, BackOffice recommends using one of the System Credential posting options.
  • System Credential Background Post - Posts a request in the background using the stored credentials entered on the Posting and File tab on the Parameters page.
  • System Credential Post Later - Posts a request at a scheduled time using the stored credentials entered on the Posting and File tab on the Parameters page.
  • Post Later – Schedules a post to run at a later date and time and in the background. There are two primary reasons to post in the background: There is too much data to post in the foreground without timing out the session, and to avoid putting too much stress on the system.

To post the Request:

  1. Click Request (Post) button.
  1. Click Post , a confirmation window displays.
  1. Click the OK button.
  1. Enter TrainingXX in Back End User ID field.
  1. Enter trainingXX in Back End Password field.
  1. Enter trainingXX in Confirm Back End Password field.
  1. Click the Authenticate button.
