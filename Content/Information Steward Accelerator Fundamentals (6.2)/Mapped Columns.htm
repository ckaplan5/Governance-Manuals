Mapped Columns

  1. Add Data Entry Controls – Column Properties

Data entry controls can be added to control which columns display in the generated data entry spreadsheet and on the Data Entry or Mass Changes pages by either adding Column Properties or configuring it at the Template Role Level for a Request based on the Template.

Column properties ensure that specific values are selected through a list box or combo box when manually entering data or specifying required fields. If the data for the request is imported from a view and the value of the field does not meet the criteria of the column property, the record displays as invalid.

Some effective column properties to control data entry are:

  • Control Status is a Column Property for various Control Types that determines how or if the column displays on a page. There are three options:
  • Enabled – Displays the column on the page. This is the default value.
  • Hide – Hides the column on the page.
  • Disabled – Protects the column. Displays as read-only columns when in edit-mode.
  • Required Fields:  There are two types of required fields:
  • Hard Required – The record cannot be saved until the field is populated.
  • Soft Required – The record can be saved but is not considered valid until the field is populated.
  • List Box:  Displays a field as a list box with options from a selected Table or View.
  • Combo Box:  Displays as a text box with options from a selected Table or View that display once text is entered in the field. To improve performance, use a combo box instead of a list box when the list view contains too many records to display suitably as a list box.
  • Check Box:  Displays as a check box that can be enabled or disabled and apply the correct format and value in the database.
