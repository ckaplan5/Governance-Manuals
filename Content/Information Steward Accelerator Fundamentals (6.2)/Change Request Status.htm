3.10 Change Request Status

Once the Project Distributions and Rules are configured in ISA. The ISA rules need to be configured on a Template in dspCompose. Multiple rules can be added to any Template and will generate requests based on the rule from ISA.

Existing Templates cannot be modified if there is one or more requests. When the Update Postal Code request was submitted, the Request Status defaulted to Request in Process. Once finished, the request is set to Completed. In order to modify the Template with the new ISA rules, the request needs to be deleted or canceled by changing the Request Status.

A canceled request can be archived. A deleted request cannot be archived and is deleted from the system once the service page runs. By default, this service page runs once daily. The service page can also be run manually to remove any requests when testing.

Requests are archived based on the Request Status. The delete archive feature permanently removes archived requests from the system and is configurable on the Vertical View of the Template and in the defaults under Configuration.

Change the Request Status for the Update Missing Postal Codes for TrainingXX to Delete.

To delete a request:

  1. Click the BackOffice Site link.
  2. Click dspCompose on the Navigation pane.
  3. Click Requests on the Navigation pane.
  1. Locate Update Missing Postal Codes for TrainingXX.
  1. Click Vertical View .
  1. Click Change Status tab.
  1. Click Delete Request , a confirmation displays.
