3.3 Configure ISA Project Distributions

Collect takes a snapshot of the registrations in IS. ISA then distributes the data to groups of users. These user groups, called Project Distributions, can receive the data as an Excel attachment (.xlsx) or can view a summary of the rule’s data.

The distributions must be configured in ISA. Projects must have at least one Distribution, but can have multiple Distributions. Each Distribution has a Distribution Owner, who is the primary contact of information for the data quality rules configured in the Distribution. Project Distributions, along with users, have rules. These rules have associated Quality Dimensions by which rules are grouped.

Once the distributions are set up and executed in ISA, the Template can be configured with the ISA rule to generate a request based on the Template.

Users will understand how to:

  • To add users to ISA Project distributions list to generate workflows to the users responsible for the failed records.
  • The two ways to add the IS rules that were imported into ISA to a distribution list.

To create a distribution:

  1. Click the BackOffice Site link.
  1. Click Information Steward Accelerator on the Navigation pane.
  2. Click Project Summary on the Navigation pane.

NOTE:The two projects from IS display in ISA.

  1. Locate ISA Training in Parent pane.
  2. Click Distributions.
