3.0 Connect ISA to Information Steward

Before configuring an ISA report to generate a request the settings need to be configured and objects created in IS and Central Management Console (CMC).All of the steps must be performed for the ISA to access and pull data from Information Steward. This is a onetime process.

For the purpose of this training the following connections have already been configured.

Use CMC to:

  • Create projects
  • Create a Connection to the ISFailedData Database that will store the Data that failed the IS Rules in Central Management Console

Use Information Steward to:

  • Create tables for connections
  • Import tables
  • Create views
  • Create rules
  • Bind the rules to tables or views
  • Run the rules

Use the System Administration component of the DSP to:

  • Add User
  • Create an ISA Security Role
  • Assign a Key Value of Target -dgRepository_IS to the Security Role
  • Add WebApp groups to the security role
  • Add Users to the Security Role
  • Set the Connection information for the ISSAP data source

Use the Common, which is a collection of utilities used throughout the DSP™, to:

  • Add an SAP Data Services Definition

Use the Collect, which is a data repository management component that is a core entity of DSP, to:

  • Set Data Services ID for the dgRepository_IS target
  • Set the Data Services Connection information
  • Build and Refresh tables
  • Refresh the Tables in Collect if Projects, Rules, or Rule Bindings are Added or Updated in IS

Use the ISA to:

  • Add a Project Distribution
  • Activate Users in a Project Distribution
  • Process the Rules

After logging into ISA, users will understand how to connect ISA to Information Steward and test the connection.

To test the connection:

  1. Return to DSP.
  1. Click Admin on the Navigation pane.
  2. Click Data Sources on the Navigation pane.
  3. Locate ISSAP.
  4. Click the Vertical View for ISSAP.
  5. Click Test Connection on the Page Toolbar, a message displays the connection passed.
