Requests are generated from a Rule in the ISA and are based on a Template configured in dspCompose. Templates are configured with one or more ISA reports by adding an ISA External Scenario. They are configured to remediate failed data from IS rules using dspCompose™.

During the request creation process, the report data is copied to a staging table where dspCompose™ maps the columns and runs any custom rules (registered as template-level events) on the data. The data is then moved to the Data Entry table for the dspCompose™ Template, and can be corrected by the first Data Entry role to begin the request posting process.

Upon completion of the following exercises, the user will understand how to:

  1. Connect ISA to Information Steward
  2. Run rules in IS.
  1. Configure the ISA to distribute failed data to users in project distributions.
  1. Create an External Request Scenario for an Information Steward-initiated Request in dspCompose™.
  1. Map Columns from the Template to Columns from the ISA Report.
  1. Create Custom Rules for Information Steward-initiated Requests, if needed.
  1. Create a Request in dspCompose™ From the ISA Report Data Viewer Page.

The purpose of these exercises is to add two rules to the Project in ISA and modify an existing Template to generate Requests for the new rules.
