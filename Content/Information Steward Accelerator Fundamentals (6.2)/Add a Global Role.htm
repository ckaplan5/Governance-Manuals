5.11 Add a Global Role

Three global roles are automatically added to the Template; Data, Review and Post. They can be modified and/or copied to a Template either manually or automatically if the Auto Copy option is selected on the Vertical View of the Roles page. There are four Role Types:

  • Data – Users enter request data by manually entering or editing data, performing a mass change, or importing data.
  • Review – Users review the data entered by the Data role, and approve or reject the data.
  • Post – Users post the approved data to SAP.
  • External Data – Receive a spreadsheet generated by dspCompse™, enter request data, and email the spreadsheet to dspCompose™. This data, once it passes validations, becomes the basis for a request. This is not supported in this version of dspCompose.

If the parameters Add Roles On Template Creation (set on the Parameters page) and Auto Copy (set on the Roles page’s Vertical View) for the global role are enabled, these roles are automatically registered to a new Template. These options are enabled by default.

Additional global roles can be added and copied to one or more Templates. If a global role is copied to a Template and a user modifies that role at the Template level, these changes will not be overridden by global role settings. If a user deletes a global role at the Template level that has the Auto Copy option selected at the global role level, the global role will not be added back to the Template.

Roles can also be created and added directly to a Template even if they are not global roles. Adding a Role to the Template does not add it to the Global Roles. Global Roles can be added to any Template created. Only the user who created the new Role has security for the Role and Template.

Once a Template is created, roles can be added or deleted as necessary.

The creator of the Template is automatically granted security to all roles for the Template. Additional users can be granted security to the Template by the Template creator. Any other users who should be assigned to the Template role must be granted access on the User Template Role Access page, accessible from the Navigation pane by selecting Configuration > Users > Templates.
