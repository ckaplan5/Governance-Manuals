NOTE:Advanced users such as BackOffice consultants should configure the comparison approval feature as it requires knowledge of both SQL and SAP. Once configured, business end users assigned to the Review role can use the feature.

To configure and use comparison approvals:

  1. Create the Template
  2. Configure the Comparison Approval Setting
  3. Modify the ComparIns Stored Procedure
  4. Register the CompareIns Stored Procedure
  5. Create a Request for Comparison Approval
  6. Use the Comparison Approval Page

When a Template is generated, dspCompose™ creates a table, a view and a stored procedure in the cMass_Data database to control the Comparison page. The naming convention for the stored procedure is web[Template Name]CompareIns. To see the before and after data entries, the CompareIns stored procedure must be modified.

The dgSAP database is a copy of tables in SAP. The stored procedure is used to insert what is currently in SAP from the dgSAP tables to the Comparison staging table in cMass_Data for the request. The dgSAP tables are a copy of SAP and loaded using Collect.

To configure comparison approval:

  1. Click Vertical View for the Review role.
  1. Click Approve and Finish Settings tab.
  1. Click Use Comparison Approval Page check box to enable it.
