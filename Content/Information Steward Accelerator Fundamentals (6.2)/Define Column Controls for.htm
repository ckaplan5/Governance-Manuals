5.18 Define Column Controls for Roles

Column display can be designates on a Template for each role. The Control Status for a column in a Template can be set to Enabled, Disabled or Hidden based on the role.

This feature works in conjunction with column properties set at the page level in the platform. dspCompose™ uses the most restrictive setting when displaying columns. In general, the Hidden setting for a column is more restrictive than a Disabled setting and a Disabled setting is more restrictive than an Enabled setting. For example, if a column is hidden at the page level in the platform but is enabled at the Template- role level in dspCompose™, the column will be hidden. This will be discussed more in a future exercise.

On the Template (Role) page, the Columns icon is disabled after a user records a Template. Once the Template is generated, these icons are enabled. To generate a Template, on the Templates page’s Vertical View, click Generate. If the user creates a Template with roles that have custom pages assigned, the Columns icon on the Template (Role) page is disabled for those roles with custom pages even after the Template is generated.

Designate which columns are enabled or disabled for the Data Role and the Manufacturing Role.

To define column controls:

  1. Click the BackOffice Site link.
  2. Click dspCompose on the Navigation pane.
  3. Click Team on the Navigation pane.
  4. Click Templates for Material Master.
  5. Locate MaterialMasterUpdateXX.
  6. Click Roles .
  1. Click Columns for the Data Role.
  1. Click Disable for the following Column Descriptions.
  • Plant
  • Manufacturer Part Number
