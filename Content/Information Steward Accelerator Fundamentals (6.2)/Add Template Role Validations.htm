5.20 Add Template Role Validations

Role validations allow for template-specific data standards to be applied at both the role and the request level. dspCompose™ runs validations registered to roles when a user assigned to that role enters a record for the request, or when a user clicks Validate for a role on the Request (Roles) page.

Role validations are views stored in a data source, typically the cMass_Data database. The view must exist in the database before the role validation can be added to the.

Validations related to list boxes, check boxes, and required fields can also be triggered when validating a role, if certain parameters are enabled.

A user can also:

  • Import validations from the registered page
  • Register validations from any pages in any component

Validation Requirements:

  • Page-level validations run against the pages to which they are registered and tie registration failures to that page. Role-level validations are tied to the page assigned for the template role.
  • All registered validation views must contain the Request ID and ID columns from the Data Entry page for the Template.
  • To add validations to a role at the template level, the Template must not be active.
  • Use the naming convention webXXX_YYYYVal where XXX is the primary table or page and YYYY describes what the validation rule is validating.

Create a validation rule in SQL to validate Manufacturer Part Number is not missing and register it to the Manufacturing Role at the Template level.

To create and register a validation rule:

  1. Open Management Studio.
  1. Locate cMass_Data database.
  1. Expand cMass_Data database.
  1. Right click Views.
  1. Select Create New View.
  1. Click ttMaterialMasterUpdateXX table.
  1. Click the Add button.
  1. Close the window.
  1. Add the following to the view:
  • ID
  • Request ID
  • MATNR
  • MFRPN
  1. Filter MFRPN with ‘’ and NULL.
